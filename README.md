# 拯救康貝特大作戰

以 **HTML5 Canvas** 打造的橫向闖關＋**彈弓物理（Matter.js）**小遊戲，主題圍繞 **康貝特 200P** 的逆襲劇情：開場與過場動畫、Boss 戰、階段轉場與結算畫面皆在同一個網頁內完成。原始碼拆成多個 `<script>` 檔依序載入（維持單一 global scope），方便維護與改關卡。

## 環境需求

- **現代瀏覽器**（Chrome / Safari / Firefox / Edge 等）。
- **本機 HTTP 伺服器（強烈建議）**：直接用 `file://` 開 `index.html` 有時會影響影片、音效或資源載入；若異常請改用下方指令。
- **網路**：預設透過 CDN 載入 [Matter.js](https://github.com/liabru/matter-js)（見 `index.html`）。

---

## 快速開始（本機遊玩）

在專案**根目錄**執行：

```bash
python3 -m http.server 5173
```

瀏覽器開啟：<http://localhost:5173/>

埠號可自訂，例如 `8080`。

## 線上部署（GitHub Pages）

- 專案內已使用 `<base href="./">` 並附一小段腳本，緩解 **repo 子路徑**（網址末尾無 `/`）時 `assets/` 相對路徑錯位的問題。
- 將整個 repo（含 `index.html`、`game/`、`assets/`、`style.css`）部署為網站根目錄或專案子路徑即可；若仍有資源 404，請確認網址是否應以 **`/` 結尾** 或改用 Pages 設定的 **Project site base path**。

## 操作說明

### 鍵盤

| 功能 | 按鍵 |
|------|------|
| 左右移動 | `←` `→` 或 `A` `D` |
| 跳躍 | `↑` 或 `W` 或 `Space` |
| 衝刺 | `Shift` 或 `J` |
| 暫停／繼續 | `P` 或 `Esc` |
| 重新開始 | `R` |
| 跳到下一關（若有支援） | `N` |
| 切換主音量（BGM／音效） | `M` |
| 確認／開始／略過部分段落 | `Enter` |

### 滑鼠／觸控

- **點畫面**：依目前狀態可用於開始遊戲、重開、略過部分過場或死亡插播等（與流程綁定）。
- **彈弓階段**：拖曳瞄準、放開發射（亦支援觸控軌跡）。
- **手機**：符合條件時會顯示左／右、跳、衝等虛擬按鈕；版面預設為橫向遊玩（見 `style.css`）。
- **螢幕角落**：遊戲內 **聲音開關** 可點擊切換。

### 開發／測試用 HUD（畫面上）

- **跳段**：快速跳到後續流程（測試用）。
- **破雙霸**：與 Boss／階段相關的測試捷徑（實際標題以畫面為準）。

## 聲音與語音

- **BGM／音效**：由 `M` 或畫面切換控制（程式內 `audio.enabled`）。
- **語音旁白（瀏覽器 Web Speech）**：預設 **關閉**，避免自動朗讀干擾遊玩。開关在 `game/split/part-01.core.js` 的 `audio.narrationEnabled`（目前為 `false`）。若日後要開啟，設為 `true` 即可恢復 `speakSubtitleNarration` 行為。

---

## 遊戲流程（階段順序）

常數 **`SLINGSHOT_FIRST_ORDER`**（於 `part-01.core.js`）決定「彈弓關」與「橫向關」在 UI 上的第一／第二階段稱呼與銜接順序。

- **預設為 `true`**：玩家會先經歷 **彈弓／拆牆（含過場動畫）**，再進入 **橫向關卡與 Boss** 等內容。
- 實際關卡文字與轉場以遊戲內提示為準。

## 專案結構

```
new-chat/
├── index.html          # 入口：canvas、觸控層、過場影片、script 載入順序
├── style.css           # 版面、手機橫向、HUD、影片遮罩等
├── assets/marketing/   # 圖片、過場影片等素材
├── game/
│   ├── README.md       # 拆檔分工說明（開發者）
│   └── split/
│       ├── part-01.core.js   # 核心狀態、音效、關卡資料、Stage2 物理、過場常數…
│       ├── part-02.stage.js   # Stage1／Boss、世界更新、過場狀態機
│       ├── part-03.render-loop.js  # 繪製、主迴圈、UI／動畫畫面
│       └── part-04.input-boot.js   # 鍵盤／滑鼠／觸控、啟動流程
├── tools/              # 可選：Playwright 場景截圖腳本（見 tools/README.md）
└── README.md           # 本檔（使用者與協作者入口）
```

載入順序 **不可任意調換**：後面的檔案依賴前面宣告的全域函式與狀態。

## 素材與快取

- **行銷／角色／產品去背圖**等放在 `assets/marketing/`。
- **`index.html`** 中 CSS／JS 使用 **`?v=…`** 查詢字串做 **cache busting**；改版記得更新版本字串，避免瀏覽器沿用舊腳本。

---

## 開發與維護

- 本專案刻意維持 **非 ES modules** 的傳統 `<script>` 串接，以降低一次性重構風險。
- 修改遊戲邏輯時請先對照 **`game/README.md`** 各檔職責，避免在錯檔修改。
- 本地若使用 Playwright 等工具產生畫面截圖，輸出預設可放在 `output/`（已列入 `.gitignore`，不進版控）。
